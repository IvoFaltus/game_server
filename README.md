# Python Game - Socket Quiz Server

A small TCP socket server that serves AI-generated multiple-choice questions. Clients connect over TCP, ask for a question on a topic, and then answer it. The server uses a password to fetch an API key from `bestapi.cz` and then calls the OpenAI Responses API to generate the question and answers.

## Features
- TCP server with a simple text protocol
- Per-client question state
- Threaded client handling with basic thread monitoring
- Colored server logs

## Requirements
- Python 3.9+
- Packages: `requests`, `colorama`

## Setup
1. Create and activate a virtual environment (optional but recommended).
2. Install dependencies:

```bash
pip install requests colorama
```

## Configuration
The server reads `host` and `port` from `config.json` unless provided via CLI flags.

Example `config.json`:
```json
{
  "host": "127.0.0.1",
  "port": "7777"
}
```

## Running the Server
The server requires a password (used to retrieve an API key from `bestapi.cz`).

```bash
python server.py --password YOUR_PASSWORD
```

Optional overrides:
```bash
python server.py --host 127.0.0.1 --port 7777 --password YOUR_PASSWORD
```

## Client Protocol
Connect to the server using any TCP client (for example `telnet` or `nc`).

Commands:
- Request a question: `q:<topic>`
- Answer a question: `a:<your_answer>`

Example session:
```
q:python
1) ...
2) ...
3) ...
a:...
```

Notes:
- Only one active question per client at a time.
- The correct answer is option `3` (as generated by the API).

## Files
- `server.py`: TCP server and client handling
- `api.py`: API key retrieval + OpenAI request
- `config.json`: Default host/port configuration
- `threads.log`: Periodic thread count log

## Troubleshooting
- If you see `Invalid IP address` or `Invalid port`, check `config.json` or CLI flags.
- If you see `wrong password`, verify the password for the API key service.
